# 百度统计埋点功能教程

本项目使用 Vue 3 自定义指令实现百度统计埋点功能。

## 原理概述

```
┌─────────────────────────────────────────────────────────────┐
│                      百度统计埋点流程                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. index.html 加载百度统计 SDK (仅正式环境 www.cxhf.com)     │
│                           ↓                                  │
│  2. main.ts 注册全局指令 v-baidu-track                       │
│                           ↓                                  │
│  3. 组件使用 v-baidu-track 指令绑定埋点                       │
│                           ↓                                  │
│  4. 用户点击元素 → 指令触发 → 调用 _hmt.push(['_trackEvent']) │
│                           ↓                                  │
│  5. 数据上报至百度统计后台                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 核心文件

| 文件 | 作用 |
|------|------|
| [index.html](index.html) | 初始化 `_hmt`，加载百度统计 SDK |
| [src/main.ts](src/main.ts) | 注册全局指令 `v-baidu-track` |
| [src/utils/baiduTrack.js](src/utils/baiduTrack.js) | 埋点指令核心实现 |

## 一、HTML 层接入

在 `index.html` 中，百度统计 SDK 仅在正式环境自动加载：

```html
<script>
  var _hmt = _hmt || [];
  (function() {
    // 仅正式环境域名才开启百度统计
    if (window.location.host == 'www.cxhf.com') {
      var hm = document.createElement("script");
      hm.src = "https://hm.baidu.com/hm.js?***你的api***";
      var s = document.getElementsByTagName("script")[0];
      s.parentNode.insertBefore(hm, s);
    }
  })();
</script>
```

**注意**：如果需要切换百度统计账号，只需修改 `hm.js` 后面的 ID 参数。

## 二、全局指令注册

在 `src/main.ts` 中注册指令：

```typescript
import baiduTrack from './utils/baiduTrack.js'

export function createApp() {
  const app = createSSRApp(App);

  // 注册全局指令
  app.directive('baidu-track', baiduTrack);

  return { app };
}
```

## 三、指令使用方式

### 基础用法

在模板中直接使用 `v-baidu-track` 指令：

```vue
<view v-baidu-track="{ category: '科大硅谷', name: '查看详情' }">
  点击我上报埋点
</view>
```

### 指令参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `category` | string | 元素 `data-category` 或 "通用按钮" | 页面/分类 |
| `action` | string | "点击" | 事件动作（暂未使用） |
| `name` | string | 元素 `data-name` 或元素文本 | 事件名称 |
| `value` | number | 0 | 数值（用于统计） |

### 使用优先级

```
指令传参 > 元素 data-* 属性 > 默认值
```

### 完整示例

```vue
<template>
  <!-- 完整参数 -->
  <button v-baidu-track="{ category: '活动页', action: '点击', name: '立即报名', value: 1 }">
    报名
  </button>

  <!-- 省略 action（使用默认值"点击"） -->
  <view v-baidu-track="{ category: '首页', name: '轮播图' }">
    轮播内容
  </view>

  <!-- 仅传 name，其他使用默认值 -->
  <div v-baidu-track="{ name: '提交按钮' }">提交</div>

  <!-- 无参数，使用元素自身信息 -->
  <div v-baidu-track>使用元素文本作为name</div>
</template>
```

### 在循环列表中使用

```vue
<view v-for="(item, index) in list" :key="index">
  <view
    v-baidu-track="{ category: '商品列表', name: item.name }"
    @click="goDetail(item)"
  >
    {{ item.name }}
  </view>
</view>
```

### 使用 data-* 属性

当多个元素共享同一 category 时，可以在元素上设置 data 属性：

```vue
<view data-category="科大硅谷" data-name="详情页">
  <view v-baidu-track>自动读取 data-* 属性</view>
</view>
```

## 四、指令实现原理

`src/utils/baiduTrack.js` 核心逻辑：

```javascript
export default {
  // 元素挂载时绑定事件
  mounted(el, binding) {
    const handleClick = () => {
      // 1. 获取用户信息
      const userInfo = useUserStore()?.getUserInfo;
      const userName = userInfo?.customerName || userInfo?.contactName || "未知用户";

      // 2. 解析埋点参数
      const {
        category = el.dataset.category || "通用按钮",
        action = "点击",
        name = el.dataset.name || el.innerText.trim(),
        value = 0,
      } = binding.value || {};

      // 3. 调用百度统计接口
      window._hmt = window._hmt || [];
      window._hmt.push([
        "_trackEvent",
        "h5" + category,  // 百度统计 category
        "按钮",            // 百度统计 action
        userName + "点击" + category + name,  // 百度统计 label
        value
      ]);
    };

    // 4. 同时监听 click、tap、launch 事件
    const events = ["click", "tap", "launch"];
    events.forEach((eventType) => {
      el.addEventListener(eventType, handleClick);
    });

    // 5. 缓存事件处理器，用于卸载时移除
    el._trackHandler = handleClick;
    el._trackEvents = events;
  },

  // 元素卸载时解绑，防止内存泄漏
  unmounted(el) {
    if (el._trackHandler) {
      el._trackEvents.forEach((eventType) => {
        el.removeEventListener(eventType, el._trackHandler);
      });
      delete el._trackHandler;
      delete el._trackEvents;
    }
  },
};
```

## 五、百度统计后台查看

上报的数据格式为：

```
category: "h5" + 传入category
action:   "按钮"
label:    "用户名点击分类名称"
value:    传入数值
```

示例：`window._hmt.push(["_trackEvent", "h5活动页", "按钮", "张三点击活动页立即报名", 1])`

登录百度统计后台 → 事件分析 → 即可查看各事件的触发次数和趋势。

## 六、开发调试

### 查看控制台日志

点击埋点元素后，控制台会输出：

```
v-baidu-track click triggered========= DIV {category: "活动页", name: "立即报名"}
百度埋点上报： { userName: "张三", category: "活动页", action: "点击", name: "立即报名", value: 1 }
```

### 本地调试说明

- 本地开发环境（localhost）不会加载百度统计 SDK
- 埋点调用会被静默忽略，不会影响正常功能
- 调试时可观察控制台输出确认埋点是否被触发

## 七、新项目迁移指南

### 1. 复制核心文件

```
将 src/utils/baiduTrack.js 复制到新项目
```

### 2. 安装依赖

确保项目已安装 Pinia（用于获取用户信息）：

```bash
npm install pinia
```

### 3. 修改百度统计 ID

在 `index.html` 中替换为您自己的百度统计 ID：

```javascript
// 替换此行
hm.src = "https://hm.baidu.com/hm.js?您的统计ID";
```

### 4. 注册指令

在 `main.ts` 或 `main.js` 中：

```javascript
import baiduTrack from './utils/baiduTrack.js'
app.directive('baidu-track', baiduTrack);
```

### 5. 如果不需要用户信息

如果项目不使用 Pinia，可简化 `baiduTrack.js` 中的用户信息获取逻辑。
