# 06-常用功能与实战

## 6.1 计数器应用

### 高速计数器

普通计数器受扫描周期限制，高速计数器使用中断机制，可达 **100kHz** 级别计数。

#### 西门子 HSC 配置（S7-1200）

1. **硬件配置**：在 CPU 属性 → **高速计数器（HSC）** → 启用通道（如 HSC1）
2. **配置参数**：
   - 计数类型：频率 / 周期 / 计数
   - 计数方向：增 / 减 / 双向
   - 门控功能：软件门 / 同步门
3. **程序中读取**：

```st
// 读取 HSC1 当前计数值
"HSC1_DB".cnt[1];  // 当前值

// 在事件中处理
IF "HSC1_DB". sts[1].cwCount = TRUE THEN
    // 计数到达事件处理
END_IF;
```

#### 三菱 HSC 配置（FX5U）

FX5U 内置高速计数器，频率可达 **100kHz**（单相）或 **50kHz**（两相）：

| 计数器 | 输入点 | 类型 |
|--------|--------|------|
| Hsc0 (C235) | X0 | 单相 |
| Hsc1 (C236) | X1 | 单相 |
| Hsc2 (C237) | X2 | 单相 |
| Hsc3 (C238) | X3 | 单相 |
| Hsc4 (C239) | X0/X1 | 两相AB相 |

**编程**：

```st
// FX5U HSC 初始化（首次扫描时执行一次）
IMASK(TRUE);  // 允许高速计数器中断

// C235 计数值存放在 D8030/D8031（32位）
// 读取
hsc_value := ((D8031 << 16) OR D8030);
```

### 实战：编码器测速

使用增量式编码器输出 A/B 相脉冲，计算电机转速。

**原理**：
- 编码器每转输出 N 个脉冲（如 1024 P/R）
- 测量 1 秒内的脉冲数 = 频率（Hz）
- 转速 RPM = 频率 × 60 / 编码器线数

```st
// 西门子 SCL 示例
VAR
    encoder_hsc AT %ID100 : DInt;  // HSC1 计数值
    speed_rpm   : Int;
    pulse_per_rev : Int := 1024;   // 编码器线数
    sample_time  : Int := 1;       // 采样时间（秒）
END_VAR

// 转速计算（RPM）
speed_rpm := (encoder_hsc * 60) / pulse_per_rev;
```

## 6.2 模拟量应用

### 温度采集（PT100 + 模拟量模块）

**接线**：
```
  PT100（三线制）
  ──[ 电阻变化 ]──→ [ 模拟量模块 AI ]──→ PLC
                      输出 4~20mA
```

**换算公式**：
```
温度值 = (当前值 - 最小值) / (最大值 - 最小值) × 量程范围

例如：测量范围 -50~150°C，输出 4~20mA
当前电流 12mA → 温度 = (12-4)/(20-4) × (150+50) - 50 = 50°C
```

### 西门子模拟量处理（S7-1200）

```st
// 读取标准化值 0~27648 对应 4~20mA
raw_value := IW64;  // 模拟量输入字

// 转换为实际工程值（0~100°C）
IF raw_value > 0 THEN
    temperature := (raw_value / 27648.0) * 100.0;
ELSE
    temperature := 0.0;
END_IF;
```

### 三菱模拟量处理（FX5U 内置）

```st
// FX5U 内置 AD 转换，结果存入 SD6080/SD6081
raw_ad := FROM(K0, H3, D100, K1);  // 读取 CH1

// 标定（0~4000 对应 0~10V）
voltage := raw_ad * 10.0 / 4000.0;
```

### PID 控制

PID（比例-积分-微分）用于精确的过程控制，如恒温、恒压、恒流。

**PID 公式**：
```
输出 = Kp × 偏差 + Ki × 偏差积分 + Kd × 偏差微分
```

**PID 参数调节口诀**：
> "参数整定找最佳，从小到大顺序查；先是比例后积分，最后再把微分加。"

| 参数 | 作用 | 调节方向 |
|------|------|----------|
| Kp（比例） | 响应速度 | 越大越快，但可能振荡 |
| Ki（积分） | 消除稳态误差 | 越大消除越快，但超调 |
| Kd（微分） | 抑制超调 | 越大抑制越强，但敏感 |

## 6.3 通信应用

### Modbus RTU 通信（RS485）

Modbus 是工业领域最通用的通信协议。

**硬件连接**：
```
  PLC (RS485)              变频器/仪表 (RS485)
  ├─ A+  ────────────────── A+
  ├─ B-  ────────────────── B-
  └─ SG  ────────────────── SG（屏蔽层）
```

**三菱 FX5U Modbus RTU 编程**：

```st
// 初始化 Modbus 参数
M8411 := TRUE;  // Modbus 连接中标志

// 发送 Modbus 读指令（读取变频器频率）
// ADPRW 指令：站号1，功能码H03，起始地址H0000，读取2字
// 结果存入 D200
ADPRW(
    En := TRUE,
    D( 1 ),           // 从站号 = 1
    Cmd( H0003 ),     // 功能码 03：读保持寄存器
    ADR( H0000 ),     // 起始地址
    NUM( 2 ),         // 读取字数
    D( D200 )          // 结果存储地址
);
```

### 以太网通信

#### 西门子 S7 通信（两台 S7-1200 互联）

```st
// S7-1200 之间用 PUT/GET 指令交换数据
// 需要在双方都开启"允许使用 PUT/GET 访问"权限

// 发送端：PUT 指令
PUT(
    REQ      := start_send,    // 发送触发
    ID       := 1,              // 连接号
    ADDR_1   := P#DB1.DBX0.0 BYTE 10,  // 本地数据源
    RD_1     := P#DB2.DBX0.0 BYTE 10   // 对方存储地址
);

// 接收端：GET 指令
GET(
    REQ      := start_recv,
    ID       := 1,
    ADDR_1   := P#DB1.DBX0.0 BYTE 10,
    RD_1     := P#DB2.DBX0.0 BYTE 10
);
```

### 自由口通信（无协议）

当设备使用私有协议时，使用自由口（Free Port）自己解析。

```st
// 西门子 S7-1200 自由口发送
// 触发发送，RS485 接口发送一串数据
IF send_trigger THEN
    SEND_P2P(
        REQ      := TRUE,
        PORT      := 1,
        BUFFER   := P#DB3.DBX0.0 BYTE 20,
        LENGTH   := 20,
        DONE     := send_done,
        ERROR    := send_error
    );
END_IF;
```

## 6.4 运动控制

### 步进电机控制

步进电机通过脉冲信号控制，每接收一个脉冲转动一个步距角。

| 参数 | 说明 |
|------|------|
| 脉冲频率 | 决定转速（频率越高转速越快） |
| 脉冲数量 | 决定转角（数量越多转角越大） |
| 方向信号 | DIR，决定正反转 |

**PTO 脉冲输出（西门子 S7-1200）**：

```st
// 配置脉冲发生器
CTRL_HSC_PTO(
    HSC_ID   := 1,       // PTO1
    AUTO_ON  := FALSE,   // 手动控制
);

// 发送脉冲序列
POSIZING_CTRL(
    HSC_ID   := 1,
    POSITION := 10000,   // 目标位置（脉冲数）
    VELOCITY := 1000,   // 速度（脉冲/秒）
    START    := start_move,
    DONE     := move_done,
    ERROR    := move_error
);
```

### 伺服电机控制

伺服电机通常使用 **位置控制模式**，接收脉冲列指令。

**常见控制模式**：

| 模式 | 控制方式 | 应用 |
|------|----------|------|
| 位置模式 | 脉冲列 / 总线 | 点位运动 |
| 速度模式 | 模拟量 / 总线 | 恒速运行 |
| 转矩模式 | 模拟量 | 张力控制 |

## 6.5 报警与安全

### 报警系统设计

```
  故障检测 → 故障标志 → 声光报警 → 停机处理
                  ↓
             故障记录
```

**报警处理程序块（伪代码）**：

```st
// 故障汇总
alarm_1 := sensor_fault OR conveyor_overload;
alarm_2 := temperature_high OR pressure_high;

total_alarm := alarm_1 OR alarm_2;

// 声光报警
buzzer := total_alarm;
alarm_light := total_alarm;

// 故障停机
IF total_alarm THEN
    motor_stop := TRUE;
    reject_valve := TRUE;  // 剔除次品
END_IF;

// 故障信息记录
IF alarm_1 AND NOT last_alarm_1 THEN
    alarm_log[alarm_index] := 'Sensor or conveyor fault';
    alarm_index := alarm_index + 1;
END_IF;
last_alarm_1 := alarm_1;
```

### 安全电路

工业设备必须设计安全电路，常见方式：

**① 急停电路（硬件互锁）**：
```
  急停按钮（常闭）──┬── 安全继电器 ── 驱动回路
                  │
             门限位开关 ─┘
```
- 急停按下 → 安全继电器断开 → 所有输出强制失电
- 这是硬件级保护，不依赖程序

**② 程序安全**：
```st
// 每次扫描都检查急停状态
IF emergency_stop = FALSE THEN
    all_outputs := FALSE;
    emergency_flag := TRUE;
END_IF;
```

## 6.6 实战项目：变频器控制电机

### 控制要求

1. 启动/停止控制（通过 PLC 输出）
2. 频率设定（通过模拟量或通信）
3. 运行频率显示（通过模拟量或通信）
4. 故障检测与处理

### 接线图

```
  PLC                    变频器
  Q0.0 ──[继电器]──────────→ STF（正转启动）
  Q0.1 ──[继电器]──────────→ STR（反转启动）
  Q0.2 ──[继电器]──────────→ RES（故障复位）
  
  D/A 模块 ──[ 0~10V ]────→ 2→5（频率设定）
  A/D 模块 ──[ 0~10V ]────→ AM（频率输出监视）
  
  RA1/RA2 ──[故障输出继电器]──→ PLC 输入 X0
```

### 完整程序（SCL 示例）

```st
// 电机启停与变频器控制
VAR
    start_btn  AT %I0.0 : BOOL;   // 启动按钮
    stop_btn   AT %I0.1 : BOOL;   // 停止按钮
    fault_in   AT %I0.2 : BOOL;   // 变频器故障输入
    run_output AT %Q0.0 : BOOL;  // 运行输出
    freq_out   AT %QW64 : INT;    // 频率设定（D/A输出）
    
    start_flag : BOOL;            // 启动标志
    freq_cmd   : INT := 0;        // 频率命令值（0-4000对应0-50Hz）
    run_cmd    : BOOL;            // 运行命令
END_VAR

// 启停逻辑
start_flag := (start_btn OR start_flag) AND NOT stop_btn AND NOT fault_in;
run_cmd := start_flag;

// 频率设定：输入0~50Hz
freq_cmd := MIN(freq_cmd + 10, 4000);  // 频率逐步上升
freq_out := freq_cmd;

// 输出控制
run_output := run_cmd;
```

## 6.7 调试技巧

1. **分段调试**：先调手动程序，确认 IO 正常后再调自动程序
2. **强制变量**：在线时强制某个 IO 点为 ON/OFF，验证接线
3. **趋势图**：用编程软件的趋势/曲线功能观察模拟量变化
4. **日志记录**：重要动作写入数据块，便于故障追溯
5. **状态机**：复杂逻辑用状态机方式编写，条理清晰
