# 永洪看板 Demo：企业综合经营分析大屏

## 一、项目概述

### 1.1 需求说明

使用永洪 BI Desktop 制作一套企业综合经营分析大屏，包含：

- **核心 KPI**：销售额、订单数、客户数、利润率等指标卡
- **销售分析**：月度趋势、区域对比、产品占比
- **客户分析**：客户等级分布、TOP 客户排行
- **实时监控**：最新订单滚动、异常预警

### 1.2 数据模型

```
数据源：MySQL / PostgreSQL

事实表：
  fact_sales        -- 销售事实表（核心）
  fact_orders       -- 订单事实表

维度表：
  dim_date          -- 日期维度（年/季度/月份/周）
  dim_region        -- 区域维度（省/市/区）
  dim_product       -- 产品维度（类别/品牌/型号）
  dim_customer      -- 客户维度（等级/行业/规模）
  dim_channel       -- 渠道维度（线上/线下/分销）
```

### 1.3 示例 SQL

```sql
-- 建表语句示例（MySQL）
CREATE TABLE dim_date (
    date_id DATE PRIMARY KEY,
    year INT,
    quarter INT,
    month INT,
    week INT,
    year_month VARCHAR(7),
    day_of_week INT
);

CREATE TABLE dim_region (
    region_id INT PRIMARY KEY,
    province VARCHAR(50),
    city VARCHAR(50),
    region_type VARCHAR(20)  -- 华北/华东/华南...
);

CREATE TABLE dim_product (
    product_id INT PRIMARY KEY,
    product_name VARCHAR(100),
    category VARCHAR(50),    -- 手机/电脑/家电
    brand VARCHAR(50),
    price DECIMAL(10,2)
);

CREATE TABLE dim_customer (
    customer_id INT PRIMARY KEY,
    customer_name VARCHAR(100),
    customer_level VARCHAR(10),  -- A/B/C/D
    industry VARCHAR(50),       -- 行业
    is_new TINYINT              -- 是否新客户
);

CREATE TABLE fact_sales (
    id INT PRIMARY KEY AUTO_INCREMENT,
    order_date DATE,
    region_id INT,
    product_id INT,
    customer_id INT,
    channel VARCHAR(20),
    quantity INT,
    unit_price DECIMAL(10,2),
    cost DECIMAL(10,2),
    amount DECIMAL(12,2),      -- 销售额
    profit DECIMAL(12,2),      -- 利润
    FOREIGN KEY (region_id) REFERENCES dim_region(region_id),
    FOREIGN KEY (product_id) REFERENCES dim_product(product_id),
    FOREIGN KEY (customer_id) REFERENCES dim_customer(customer_id)
);
```

## 三、KPI 指标卡设计

### 3.1 组件配置

使用「指标卡」组件，绑定数据集中的聚合值：

**数据集 SQL：**

```sql
SELECT 
    SUM(amount)   AS total_amount,     -- 销售额
    COUNT(id)    AS order_count,      -- 订单数
    COUNT(DISTINCT customer_id) AS customer_count,  -- 客户数
    SUM(profit)/SUM(amount)*100 AS profit_rate  -- 利润率
FROM fact_sales
WHERE order_date = CURRENT_DATE
```

**指标卡属性配置：**

```
┌─────────────────────────────────────┐
│  💰 今日销售额                        │
│                                     │
│  ¥12,845,678                        │
│                                     │
│  ↑ 12.3%  vs 昨日                   │
└─────────────────────────────────────┘

主数值：
  绑定字段：total_amount
  格式：货币 ¥#,##0
  字号：36pt，加粗，主题色

副数值（同期比）：
  公式：(今日-昨日)/昨日*100
  格式：0.0%
  颜色：正数绿色↑，负数红色↓
```

### 3.2 指标卡布局

使用「卡片容器」布局四个指标卡：

```
┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│  销售额   │ │  订单数   │ │  客户数   │ │  利润率   │
│  ¥1284万  │ │  5,678   │ │  1,234   │ │  23.5%   │
│  ↑12.3%  │ │  ↑8.5%   │ │  ↑5.2%   │ │  ↑2.1%   │
└──────────┘ └──────────┘ └──────────┘ └──────────┘
```

## 五、趋势组合图

### 5.1 组件选型

使用「组合图」组件，同时展示折线（趋势）和柱状（对比）：

```
X轴：月份（1-12月）
左Y轴：销售额（柱状图，蓝色）
右Y轴：同比增长率（折线图，红色）
```

### 5.2 绑定数据

**数据集 SQL：**

```sql
SELECT 
    d.year_month AS month,
    SUM(f.amount) AS amount,
    SUM(f.profit)/SUM(f.amount)*100 AS profit_rate,
    -- 去年同期
    LAG(SUM(f.amount)) OVER(ORDER BY d.year_month) AS last_year_amount
FROM fact_sales f
JOIN dim_date d ON f.order_date = d.date_id
GROUP BY d.year_month
ORDER BY d.year_month
```

**绑定配置：**

```
系列1（柱状）：amount（销售额）
系列2（折线）：profit_rate（利润率）
系列3（柱状）：last_year_amount（去年同期销售额，灰色半透明）
```

## 六、客户排行榜

### 6.1 轮播列表组件

使用「轮播列表」组件，自动滚动播放 TOP 10 客户：

**数据集 SQL：**

```sql
SELECT 
    customer_name,
    SUM(amount) AS total_amount,
    COUNT(DISTINCT order_id) AS order_count,
    SUM(profit)/SUM(amount)*100 AS profit_rate
FROM fact_sales
GROUP BY customer_name
ORDER BY total_amount DESC
LIMIT 10
```

基本设置：

├── 序号：圆形徽章，主题色

## 七、散点图分析

### 7.1 适用场景

展示客户「销售额 vs 利润」分布，发现高价值客户：

**数据集 SQL：**

```sql
SELECT 
    customer_name,
    SUM(amount) AS amount,
    SUM(profit) AS profit,
    SUM(profit)/SUM(amount)*100 AS profit_rate
FROM fact_sales
GROUP BY customer_name
HAVING SUM(amount) > 0
```

**绑定配置：**

```
X轴（散点位置）：amount（销售额）
Y轴（散点位置）：profit_rate（利润率）
点大小：order_count（订单数）
颜色：profit（利润，正绿负红）
```

### 7.2 象限分析

```
右上象限（高销售额+高利润率）：⭐ 明星客户，重点维护
右下象限（高销售额+低利润率）：⚠️ 薄利多销，关注成本
左上象限（低销售额+高利润率）：💎 潜力客户，挖掘需求
左下象限（低销售额+低利润率）：📋 待评估客户
```

    order_date,

## 九、异常预警

### 9.1 预警规则

设置业务规则，自动标记异常数据：

| 预警类型 | 规则 | 效果 |
|---------|------|------|
| 订单金额异常 | 单笔订单 > 100万 | 橙色边框 |
| 利润率为负 | profit < 0 | 红色背景 |
| 新客户大额 | 新客户 & 金额 > 50万 | 蓝色标记 |
| 区域下滑 | 区域同比 < -20% | 红色箭头 |

### 9.2 实现方式

使用「文字云」或「自定义 HTML」组件，结合「数据警示」功能：

```
条件格式化 → 添加预警规则：
├── 利润率 < 0 → 红色背景 + 🔴
├── 利润率 < 5% → 橙色背景 + ⚠️
├── 销售额 > 80万 → 高亮显示
└── 同比 < -20% → 红色 ↓ 箭头
```
