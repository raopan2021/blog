# 帆软报表 Demo：销售数据管理系统

## 一、项目概述

### 1.1 需求说明

本 Demo 展示使用帆软 FineReport 开发一套完整的销售数据管理系统，包含：

- **Dashboard**：多 KPI 指标卡 + 趋势图
- **明细报表**：带参数查询的销售订单明细
- **汇总报表**：按区域/产品/时间的分级汇总
- **排名报表**：TOP N 排行榜

### 1.2 数据模型

### 1.3 数据库建表 SQL

```sql
-- 区域维度表
CREATE TABLE dim_region (
    region_id   INT PRIMARY KEY,
    region_name VARCHAR(50),
    region_type VARCHAR(20)  -- 华北/华东/华南/华中/西南/西北
);

-- 产品维度表
CREATE TABLE dim_product (
    product_id   INT PRIMARY KEY,
    product_name VARCHAR(100),
    category     VARCHAR(50),   -- 手机/电脑/家电/服装
    unit_price   DECIMAL(10,2),
    cost         DECIMAL(10,2)
);

-- 客户维度表
CREATE TABLE dim_customer (
    customer_id   INT PRIMARY KEY,
    customer_name VARCHAR(100),
    customer_type VARCHAR(20),  -- 直客/经销商/代理
    credit_level  VARCHAR(10)   -- A/B/C/D
);

-- 销售订单事实表
CREATE TABLE fact_sales_order (
    order_id     INT PRIMARY KEY AUTO_INCREMENT,
    order_date   DATE,
    region_id    INT,
    product_id   INT,
    customer_id  INT,
    quantity     INT,
    unit_price   DECIMAL(10,2),
    discount     DECIMAL(3,2) DEFAULT 1.0,
    amount       DECIMAL(12,2),    -- 计算字段
    profit       DECIMAL(12,2),    -- 计算字段
    FOREIGN KEY (region_id)   REFERENCES dim_region(region_id),
    FOREIGN KEY (product_id) REFERENCES dim_product(product_id),
    FOREIGN KEY (customer_id) REFERENCES dim_customer(customer_id)
);

-- 创建索引
CREATE INDEX idx_order_date ON fact_sales_order(order_date);
CREATE INDEX idx_region ON fact_sales_order(region_id);
CREATE INDEX idx_product ON fact_sales_order(product_id);
```

### 2.2 数据集准备

```sql
-- 数据集1：KPI指标汇总
SELECT 
    SUM(amount)   AS total_amount,
    COUNT(order_id) AS order_count,
    COUNT(DISTINCT customer_id) AS customer_count,
    SUM(profit)/SUM(amount)*100 AS profit_rate
FROM fact_sales_order
WHERE order_date BETWEEN ${startDate} AND ${endDate}

-- 数据集2：月度趋势
SELECT 
    DATE_FORMAT(order_date, '%Y-%m') AS month,
    SUM(amount) AS amount
FROM fact_sales_order
WHERE order_date BETWEEN ${startDate} AND ${endDate}
GROUP BY DATE_FORMAT(order_date, '%Y-%m')
ORDER BY month

-- 数据集3：区域排名
SELECT 
    r.region_name,
    SUM(f.amount) AS amount,
    SUM(f.profit) AS profit
FROM fact_sales_order f
JOIN dim_region r ON f.region_id = r.region_id
WHERE f.order_date BETWEEN ${startDate} AND ${endDate}
GROUP BY r.region_name
ORDER BY amount DESC

-- 数据集4：产品类别占比
SELECT 
    p.category,
    SUM(f.amount) AS amount
FROM fact_sales_order f
JOIN dim_product p ON f.product_id = p.product_id
WHERE f.order_date BETWEEN ${startDate} AND ${endDate}
GROUP BY p.category

-- 数据集5：最新订单
SELECT 
    f.order_id,
    f.order_date,
    r.region_name,
    p.product_name,
    c.customer_name,
    f.quantity,
    f.amount,
    f.profit
FROM fact_sales_order f
JOIN dim_region r ON f.region_id = r.region_id
JOIN dim_product p ON f.product_id = p.product_id
JOIN dim_customer c ON f.customer_id = c.customer_id
ORDER BY f.order_date DESC
LIMIT 20
```

### 2.3 KPI 指标卡实现

使用 `单元格` + `插入图表` → 选择「仪表盘」或直接用指标卡组件：

**指标卡公式配置：**

```
销售额指标卡：
  数值：=ds1.total_amount
  格式：货币 ¥#,##0.00
  字号：大号28pt，加粗
  
同期比公式：
  =IFNULL((ds1.total_amount - ds1_last.total_amount) / ds1_last.total_amount * 100, 0)
  格式：0.0%
  颜色：IF(值>0, 红色, 绿色) + 上/下箭头
```

### 2.4 趋势图配置

**绑定数据：**

```
系列名：月度
分类轴（X轴）：month
系列值（Y轴）：amount
```

**图表属性：**

```
类型：折线图（带面积填充）
线条颜色：#1890ff
填充：渐变蓝色（透明度20%）
数据标签：显示数值
平滑曲线：开启
```

## 五、帆软参数传递高级用法

### 5.1 跨 Sheet 参数

在决策报表中，一个参数可以供多个组件使用：

```
参数面板（Sheet1）定义的参数
        ↓
所有 Sheet 共用（Sheet2、Sheet3...）
```

### 5.2 父子报表联动

**父报表：** 区域列表，点击区域跳转子报表

```
=REPORTIF("区域销售明细.frm", "region=" + A2)
```

**子报表：** 接收父报表参数，显示该区域明细

```
{{region}}  -- 获取父报表传递的区域参数
```

### 5.3 动态表头

根据用户选择动态显示不同的列：

```javascript
// 根据参数决定显示哪些列
${if(showProfit == "是", "利润", "")}

// 实际应用中
SELECT 订单号, 销售额 ${if(len(showProfit) > 0 and showProfit == "是", ", 利润", "")}
FROM 订单表
```

## 七、常见问题

### 7.1 报表导出 Excel 格式错乱

**原因：** 合并单元格导致

**解决：**
- 避免不必要的单元格合并
- 或在导出设置中选择「导出未合并的单元格」

### 7.2 数据量大了加载慢

**优化方案：**
- 使用参数控制数据范围
- SQL 查询避免 SELECT *
- 为大表建立索引

### 7.3 决策报表在不同分辨率下变形

**解决：**
- 使用自适应布局插件
- 或按主流分辨率（1920×1080）设计
