# 永洪 BI 开发进阶

## 一、Data Modeler 数据建模

Data Modeler 是永洪 BI 的核心 ETL 工具，支持拖拽式数据处理，无需编码。

### 1.1 进入 Data Modeler

1. 点击顶部菜单「数据」→「Data Modeler」
2. 或在数据集界面选择「创建数据模型」

### 1.2 数据模型界面

```

│  Data Modeler - 数据建模工具                                 │
├──────────┬──────────────────────────────────────────────────┤
│          │  工具栏：📥输入 | 🔄转换 | ➕输出 | ▶执行         │
│  左侧资源│──────────────────────────────────────────────────│
│          │                                                   │
│  📁 数据源 │  中间画布（节点流程图）                        │

│          │                                                   │
```

### 1.3 数据输入节点

| 节点 | 说明 |
|------|------|
| **表输入** | 从数据源读取表 |
| **SQL 输入** | 自定义 SQL 查询 |
| **动态参数** | 支持运行时参数 |
| **联合输入** | 多个数据源 Union |

### 1.4 数据转换节点

| 节点 | 说明 |
|------|------|

| **分组聚合** | GROUP BY 汇总 |
| **排序** | ORDER BY |
| **合并行** | Union / Concatenate |
| **行转列/列转行** | 数据透视 |

### 1.5 实战：订单数据清洗

**需求：** 从原始订单表清洗出月度汇总表

**步骤：**

```
1. 拖入「表输入」节点 → 选择 sales_order 表
        ↓
2. 拖入「列选择」节点 → 只保留需要的列
        ↓
3. 拖入「字段计算」节点 → 新增月份字段：DATE_FORMAT(order_date, '%Y-%m')
        ↓
4. 拖入「分组聚合」节点 → 按月份分组，SUM(amount)
        ↓
5. 拖入「排序」节点 → 按月份升序
        ↓
6. 拖入「输出表」节点 → 输出到中间表 monthly_sales
```

**字段计算表达式示例：**

```javascript
// 新增字段：销售季度
CASE 
  WHEN MONTH(order_date) BETWEEN 1 AND 3 THEN 'Q1'
  WHEN MONTH(order_date) BETWEEN 4 AND 6 THEN 'Q2'
  WHEN MONTH(order_date) BETWEEN 7 AND 9 THEN 'Q3'
  ELSE 'Q4'
END

// 新增字段：客单价
amount / customer_count

// 新增字段：同比增长率
(amount - LAG(amount) OVER(PARTITION BY region ORDER BY month)) 
/ LAG(amount) OVER(PARTITION BY region ORDER BY month) * 100
```

## 二、高级图表配置

### 2.1 联动钻取

#### 层级钻取

设置钻取路径，实现从省→市→区的下钻：

1. 选择组件 → 属性 → 交互 → 钻取
2. 添加钻取层级

```
层级1（省）→  层级2（市）→  层级3（区）
华南          →  广东省       →  广州市
              →  广西省       →  南宁市
```

#### 设置步骤

```javascript
// 1. 省份地图：绑定 region 字段
// 2. 城市柱状图：绑定 city 字段，设置父级字段为 region
// 3. 区县表格：绑定 district 字段，设置父级字段为 city

// 钻取后，柱状图自动过滤为当前选中省份的数据
```

#### 环形钻取

在同一图表上点击不同层级切换：

```
第一层：显示所有省份
↓
点击省份
↓
第二层：显示该省所有城市
↓
点击城市
↓
第三层：显示该市所有区县
```

### 2.2 聚合计算

在图表上直接进行二次计算：

| 聚合方式 | 说明 | 示例 |
|---------|------|------|
| 合计 | 总计 | 销售额总计 |
| 占比 | 占总体比例 | 销售额占比 |
| 排名 | 升序/降序排名 | 区域排名 |
| 累计 | 累计值 | 累计销售额 |
| 同期值 | 去年同期 | 去年同月 |
| 环比值 | 上期环比 | 环比增长率 |
| 移动平均 | 滑动窗口均值 | 7日均线 |

**示例：计算同比增长率**

```
数值绑定 → 右键 → 添加计算 → 同比增长率
公式: (本期值 - 去年同期值) / 去年同期值 * 100
```

### 2.3 自定义颜色

#### 使用渐变色

```javascript
// 渐变色配置示例
{
  type: 'linear',
  x0: 0, y0: 0,
  x1: 1, y1: 0,

    { offset: 0, color: '#00C853' },   // 绿色（增长）

```

```
销售额 500-1000万 → 黄色
销售额 < 500万 → 红色
```

## 四、性能优化

### 4.1 数据抽取

当数据量超过千万级别时，开启数据抽取提升查询速度：

```
数据集属性 → 抽取 → 抽取到本地
```

**抽取模式：**
| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 全量抽取 | 一次性抽取全部数据 | 数据量 < 1000万 |
| 增量抽取 | 只抽取新增数据 | 定期刷新的历史数据 |
| 实时抽取 | 数据变更实时同步 | 实时性要求高 |

### 4.2 数据库内计算

将计算下推到数据库执行，减少数据传输：

```
数据集属性 → 高级 → 勾选「数据库内计算」
```

**效果：** 数据在数据库端完成聚合后再传输，大幅减少网络流量。

### 4.3 索引优化

确保 WHERE、JOIN、GROUP BY 字段有索引：

```sql
-- 为销售表添加复合索引
CREATE INDEX idx_sales_region_date 
ON sales(region, order_date);

-- 为日期字段添加索引
CREATE INDEX idx_sales_date 
ON sales(order_date);
```

### 4.4 缓存策略

| 缓存类型 | 说明 | 配置位置 |
|---------|------|---------|
| 报告缓存 | 整页报告结果缓存 | 报告属性 → 缓存 |
| 组件缓存 | 单个组件结果缓存 | 组件属性 → 缓存 |
| 数据集缓存 | SQL 结果缓存 | 数据集属性 → 缓存 |

```
缓存时间配置示例：
├── 高频访问报告：缓存 5 分钟
├── 日报类报告：缓存 30 分钟
├── 历史数据报告：缓存 2 小时
└── 实时监控大屏：不缓存
```

### 4.5 SQL 优化检查清单

```
□ 避免 SELECT *，只取需要的列
□ WHERE 条件字段有索引
□ 避免在 WHERE 中使用函数，导致索引失效
□ 大量数据时避免 DISTINCT，用 GROUP BY 替代
□ 合理使用 LIMIT 限制数据量
□ 避免嵌套子查询，改用 JOIN
□ UNION ALL 替代 UNION（不需要去重时）
□ 复杂查询考虑用存储过程预计算
```

## 五、权限管理

### 5.1 用户与角色

永洪 BI 的权限体系：**用户 → 角色 → 权限**

```
用户（张三）→ 属于「销售部」角色 → 拥有「查看销售报表」权限
                → 属于「数据分析师」角色 → 拥有「管理数据集」权限
```

### 5.2 数据权限

#### 行级权限

限制用户只能看到自己的数据：

```sql
-- 方案一：使用用户参数过滤
WHERE sales_person = '${CURRENT_USER}'

-- 方案二：使用组织架构过滤
WHERE org_id IN (
  SELECT org_id FROM user_org WHERE user_id = '${CURRENT_USER_ID}'
)
```

#### 列级权限

限制用户只能看到部分字段：

```
设置列权限：
- 财务指标列（成本、利润）→ 仅「财务部」可见
- 人员信息列（薪资）→ 仅「HR」可见
```

## 八、常见问题解决

### 8.1 报告加载慢

| 排查步骤 | 解决方案 |
|---------|---------|
| 1. 检查数据量 | 开启数据库内计算 |
| 2. 检查 SQL 效率 | 优化 SQL，添加索引 |
| 3. 检查计算字段 | 减少复杂计算 |
| 4. 检查组件数量 | 减少单个报告组件数 |
| 5. 开启缓存 | 配置报告/组件缓存 |

### 8.2 数据不一致

| 原因 | 解决方案 |
|------|---------|
| 多数据源时间差 | 统一数据更新时间 |
| 参数默认值不同 | 检查全局参数设置 |
| 精度丢失 | 统一数值类型和精度 |
| 日期时区问题 | 统一时区设置 |

### 8.3 导出 PDF 失败

- 检查 PDF 插件是否安装
- 报告尺寸是否过大
- 是否有不支持导出的组件（如某些地图组件）
