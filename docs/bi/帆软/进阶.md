# 帆软 FineReport 开发进阶

> 深入学习帆软 FineReport 的高级特性，包括聚合报表、交互报表、性能优化、二次开发等

## 一、聚合报表与主子报表

### 1.1 聚合报表概念

聚合报表：一个页面包含多个独立报表块，各自独立扩展，互不影响。

适用场景：
- 仪表盘型报表（多个 KPI 卡片）
- 复杂报表（表头复杂+数据+图表混合）
- 对比分析（左右/上下对比）

### 1.2 聚合报表制作

1. 新建聚合报表：`文件 → 新建 → 聚合报表`
2. 添加报表块：`插入 → 报表块`
3. 各报表块独立设计

```
┌─────────────────────────────────────────┐
│              聚合报表示例                   │
├─────────────┬───────────────────────────┤
│  KPI卡片1   │      KPI卡片2              │
│  本月销售额  │     本月订单数             │
├─────────────┴───────────────────────────┤
│           销售明细表                       │
│  日期   客户   金额   地区              │
│  ...    ...    ...    ...               │
├─────────────────────────────────────────┤
│           【柱状图趋势】                   │
└─────────────────────────────────────────┘
```

### 1.3 主子报表

主子报表：一张主表带多张子表，子表跟随主表数据扩展。

**制作步骤**：
1. 主模板设置主数据集
2. 插入子模板（子报表）
3. 配置主子关联字段

```javascript
// 主表单元格配置
// A2 为主数据集的"订单号"字段
=A2

// 子报表传参
// 在子报表单元格中使用主表字段
="${ORDER_NO}"
```

### 1.4 悬浮物与报表块

```javascript
// 悬浮图表配置
// 绑定数据时使用 filter 过滤
=chart(
  SERIES,
  "销量",
  filter(ds1.groupValue("month"), ds1.sum("amount") > 10000)
)
```

## 二、深入公式与函数

### 2.1 常用高级函数

```javascript
// 跨sheet引用（多 sheet 报表）
=sheet!A1
=sheet1!B2:C10

// 条件汇总
=SUMIF(A2:A100, "华南", C2:C100)
=SUMIFS(C2:C100, A2:A100, "华南", B2:B100, ">5000")

// 排名
=RANK(C2, C2:C100)          // 升序排
=RANK(C2, C2:C100, 0)       // 降序排

// 字符串处理
=TRIM(A2)                    // 去空格
=LEN(A2)                     // 字符串长度
=LEFT(A2, 4)                // 取左边4位
=MID(A2, 5, 3)              // 取中间3位
=CONCATENATE(A2, "-", B2)  // 字符串拼接

// 日期函数
=TODAY()                    // 当前日期
=YEAR(TODAY())              // 年
=MONTH(TODAY())             // 月
=DAY(TODAY())               // 日
=DATEDIF(A2, TODAY(), "D") // 日期差
```

### 2.2 报表内 Rank 函数

```javascript
// 按分组进行组内排名
=RANK(C2, C2:C20, 0)

// 带条件分组排名
=RANK(C2, C2:C100, 0, "dept='" + A2 + "'")
```

### 2.3 层次坐标

帆软独有语法，用于引用扩展后的单元格：

```javascript
// 层次坐标基本语法
=Cell[A2:r1]

// 扩展方向
=Cell[A2:$B2]    // 列方向扩展
=Cell[$A2:A2]    // 行方向扩展
=Cell[A2:A10]    // 指定范围

// 实际应用：计算环比
=(C2 - C2[A2:-1]) / C2[A2:-1]

// 计算同比
=(C2 - C2[A2:-12]) / C2[A2:-12]
```

### 2.4 动态公式

```javascript
// 根据参数动态选择列
=SELECT(
  ${colName},        // 参数：列名
  "dept='" + A2 + "'" // 过滤条件
)

// 动态汇总列
=SUM(
  ${if(len(col) > 0, "ds1." + col, "0")}
)
```

## 四、填报与数据录入

### 4.1 填报报表概念

填报模式：用户可以在报表页面录入/修改数据，并提交到数据库。

适用场景：
- 数据采集表单
- 审批流程中的数据录入
- 简易的 CRUD 维护页面

### 4.2 填报属性设置

1. 单元格设置为"可写"
2. 设置"填报类型"（文本/数字/日期等）
3. 设置"数据校验"规则

```javascript
// 单元格填报属性
// 类型：数字
// 校验：0-100 范围
// 必填：是
```

### 4.3 数据提交配置

```javascript
// 提交类型配置
// 智能提交：根据数据库表结构自动生成
// 自定义提交：编写提交逻辑

// 自定义提交 SQL
INSERT INTO employee (name, dept, salary)
VALUES (
  ${A2},           // 姓名
  ${B2},           // 部门
  ${C2}            // 工资
)

UPDATE employee 
SET salary = ${C2}
WHERE name = ${A2}
```

### 4.4 填报校验

```javascript
// 单元格校验公式
// 校验工资 > 0

// 校验手机号格式

// 校验必填
```

## 七、性能优化

### 7.1 SQL 优化

```sql
-- ❌ 低效：SELECT *
SELECT * FROM sales WHERE 1=1

-- ✅ 高效：只取需要的字段
SELECT order_no, customer_name, amount, sale_date
FROM sales 
WHERE sale_date BETWEEN ${startDate} AND ${endDate}

-- ✅ 避免子查询，尽量用 JOIN
-- ✅ 善用 LIMIT 限制返回行数
-- ✅ 为 WHERE 条件字段建立索引
```

### 7.2 报表设计优化

| 优化点 | 说明 |
|-------|------|
| 减少单元格数量 | 控制在 1000 个以内 |
| 避免大数据集 | 用参数过滤数据范围 |
| 慎用公式 | 复杂公式会显著影响性能 |
| 图表数据 | 图表数据量控制在 1000 以内 |
| 分页加载 | 大数据量分页展示 |

### 7.3 缓存策略

```javascript
// 开启报表缓存
// 模板 → 模板 Web 属性 → 缓存设置
// 缓存时间：可设置分钟/小时

```

### 7.4 大数据量处理

```javascript
// 使用行引擎（按需加载）
// vs 列引擎（全量加载）

// 配置：服务器 → 集群 → 分页设置
```

```
├──────────┴──────────┴──────────┴──────────────────┤

│                                                   │
├─────────────────────┬────────────────────────────┤
│  地区销售排名(柱状) │   TOP5客户(条形图)           │
```

### 9.2 实现方案

1. **聚合报表**作为容器，嵌入多个报表块
2. **KPI 卡片**：用独立数据集 COUNT
3. **图表**：使用帆软内置图表或 ECharts
4. **明细表**：分页展示，配置行权限

### 9.3 关键配置

```javascript
// KPI 指标 - 使用公式计算
// 本月销售额
=SUM(ds1.select("amount", "month = MONTH(TODAY())"))

// 同比增长率
=FORMAT(
  (SUM(ds1.select("amount", "month = MONTH(TODAY())")) 
   - SUM(ds1.select("amount", "month = MONTH(TODAY())-12")))
  / SUM(ds1.select("amount", "month = MONTH(TODAY())-12")),
  "0.0%"
)

// 颜色条件显示
// 增长为正：绿色
// 增长为负：红色
=IF(C2 > 0, "green", "red")
```

## 十、常见问题与解决

### 10.1 报表模糊匹配参数失效

```sql
-- ❌ 错误写法
WHERE name LIKE '%${name}%'

-- ✅ 正确写法
WHERE 1=1
${if(len(name) > 0, "AND name LIKE '%" + name + "%'", "")}
```

### 10.2 跨库关联（多数据源）

```javascript
// 使用联邦查询
// 帆软支持跨数据源 JOIN

// 数据集配置
SELECT 
    a.order_no,
    b.product_name,
    c.category_name
FROM ds_mysql.order a
LEFT JOIN ds_oracle.product b ON a.product_id = b.id
LEFT JOIN ds_mysql.category c ON b.category_id = c.id
```
