# 换行算法与粘合规则

> Pretext 如何决定在哪个位置换行？ Knuth-Plass 算法与 Unicode Line Breaking Algorithm

## 经典的换行问题

```
给定宽度 W 和文本 T，如何将 T 切割成若干行，
使得每行都不超过 W，且整体布局"美观"？
```

这不是一个简单的问题！想象一下：

```javascript
// 文本："中华民族伟大复兴"
// 宽度：只能放 5 个字

// 方案 A（贪心算法）：
// 第一行：中华民族伟大
// 第二行：复兴
// ❌ 最后一行动太短，不美观

// 方案 B（Knuth-Plass）：
// 第一行：中华民族
// 第二行：伟大复兴
// ✅ 两行比较均衡
```

## 贪心算法 vs Knuth-Plass

### 贪心算法（简单但不美观）

```javascript
function greedyLineBreak(text, width, measure) {
  const lines = []
  let currentLine = ''

  for (const char of text) {
    const newLine = currentLine + char
    if (measure(newLine) <= width) {
      currentLine = newLine
    } else {
      if (currentLine) lines.push(currentLine)
      currentLine = char
    }
  }

  if (currentLine) lines.push(currentLine)
  return lines
}
```

问题：**每行尽可能填满，不考虑整体美观度**。

### Knuth-Plass 算法（美观但复杂）

Knuth-Plass 算法的核心思想：

1. **badness（糟糕度）**：计算某行"太满"或"太空"的程度
2. **总成本**：所有行的 badness 之和
3. **最优解**：总成本最低的换行方案

```javascript
// 糟糕度函数（简化版）
function badness(lineWidth, availableWidth) {
  const ratio = lineWidth / availableWidth
  // 越接近 1.0（填满）越理想
  // 越远离 1.0（太空或太满）越糟糕
  return Math.abs(1 - ratio) ** 3 // 立方惩罚
}
```

## Pretext 的换行策略

Pretext 实现了 **Knuth-Plass 变体**，换行考虑因素：

### 1. 断点优先级（Break Opportunity）

Unicode 定义了每个字符的断行属性：

| 字符类型 | 示例 | 断行优先级 |
|---------|------|-----------|
| 空格 | `' '` | 高（可断行） |
| 普通字符 | `'abc'` | 低（不可断） |
| 中文 | `'中'` | 高（可断） |
| 标点 | `',.'` | 视情况 |

```javascript
// Pretext 内部按 Unicode Line Breaking Algorithm 标记每个断点
```

### 2. 强制断行 vs 建议断行

```javascript
// 强制断行（\n）
const prepared = prepare('第一行\n第二行', '16px Inter')
// Pretext 会尊重 \n 的硬换行

// 软换行（由算法决定）
const prepared = prepare('中华民族伟大复兴', '16px Inter')
// Pretext 决定在哪断行
```

### 3. Hyphenation（连字符）

对于英文，Pretext 支持连字符断开：

```javascript
// 没有连字符时：
// "internationalization"
// 只能整个单词移动到下一行
// 可能导致大量空白

// 有连字符时：
// "internationa-"
// "lization"
// 更好地利用空间
```

## Pretext 的粘合规则（Glue）详解

粘合规则定义**不可分割的最小单元**：

### 1. URL 粘合

```javascript
const prepared = prepare('链接: https://chenglou.me', '16px Inter')

// 不会断成：
// 链接: https://chengl
// ou.me

// 而是整体移动或整体保留
```

### 2. 数字串粘合

```javascript
const prepared = prepare('订单号: 20260301123456789', '16px Inter')

// 避免：
// 订单号: 2026030112
// 3456789

// 数字串尽量保持连续
```

### 3. Emoji 粘合

```javascript
const prepared = prepare('中国 🇨🇳 加油 🇨🇳', '16px Inter')

// Emoji 不会在中间断开
// 每个 emoji 序列作为整体处理
```

## `whiteSpace: 'pre-wrap'` 模式

当你想让 `\n`、Tab、空格**原样保留**时：

```typescript
const textareaValue = `第一行
第二行（缩进）`

// 普通模式：多个空格合并，\n 视为可断行位置
const prepared1 = prepare(textareaValue, '16px Inter')

// pre-wrap 模式：空格和 \n 都原样保留
const prepared2 = prepare(textareaValue, '16px Inter', {
  whiteSpace: 'pre-wrap'
})
```

## 实战：自定义换行行为

```typescript
import { prepare, layout } from '@chenglou/pretext'

// 场景：窄列（只放得下 3 个字）
const narrow = prepare('中华民族伟大复兴', '16px Inter')
const result = layout(narrow, 48, 20) // 假设每个字约 16px，3*16=48

// result.lineCount = 2
// 第一行：中华
// 第二行：民族伟大复兴

// 场景：宽列（放得下 5 个字）
const wide = prepare('中华民族伟大复兴', '16px Inter')
const result2 = layout(wide, 80, 20) // 5*16=80

// result2.lineCount = 1
// 一行就够
```

## 总结

| 概念 | 说明 |
|------|------|
| Knuth-Plass | 经典的段落排版算法，追求整体美观 |
| 贪心算法 | 简单但不考虑整体美观 |
| 断点优先级 | Unicode 定义的字符断行属性 |
| 粘合（Glue） | 不可分割的字符组合 |
| badness | 衡量某行"不完美"程度的指标 |
| `pre-wrap` | 保留原始空格和换行符 |
